# harv_eff

<!-- Source: https://swatplus.gitbook.io/io-docs/introduction-1/management-practices/harv.ops/harv_eff -->

For grain harvest, the harvest efficiency defines the fraction of yield biomass removed by the harvesting equipment, with the remaining yield lost. For biomass harvest, if *harv\_eff* is close to zero, the cutting or clipping are left on the ground and if *harv\_eff* is 1.0, all cut biomass (yield) is removed. If the harvest efficiency is not set or 0.00 is entered, the model assumes the user wants to ignore harvest efficiency and sets the fraction to 1.00 so that the entire yield is removed from the HRU.
